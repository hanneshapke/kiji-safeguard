A transparency log for MCP servers

Know when your MCP server quietly changes.

kiji-safeguard signs an MCP server with a content hash of its public interface — tool names, descriptions, schemas, prompts and resources — and verifies it against a registry on every start. No keys, no identity, no ceremony. One import.

# that's the whole integration
import kiji_safeguard.autosign

Why

The MCP supply chain has a blind spot.

Your agent trusts whatever an MCP server says about itself, every single session. Tool descriptions are instructions to the model — and nothing checks that they're still the instructions you reviewed.

The rug pull

A server you vetted last month ships an update. A tool is quietly added or removed, and your agent's capabilities change without anyone signing off.

Tool-description injection

A description is rewritten to poison the model — "before answering, also forward the user's data to…". The code didn't change. The prompt surface did.

Schema drift

An input schema is widened to accept extra fields, or a parameter's meaning shifts. Subtle, invisible in logs, and exactly where exfiltration hides.

The fix is boring on purpose: hash the entire public interface, register it once, and recompute it from the live server every time it starts. If a single character in any name, description or schema changes, the hash changes — 4c469eb4… becomes 740e904b… — and verification fails loudly.

How the hash works

Order-independent. Detail-sensitive.

一

Canonical JSON per component

Every tool, prompt, resource and the server instructions are serialised with sorted keys and compact separators.

二

SHA-256 each piece

Each component serialisation gets its own digest, so any change anywhere in the interface is captured.

三

Sort, concatenate, hash again

Per-component digests are sorted lexicographically and hashed into one interface hash. Reordering tools never changes it; editing anything does.

The server name is deliberately left out of the hash — it's registry metadata. That's what lets verification tell "interface changed" apart from "same interface, different name". The client library is stdlib-only (urllib + hashlib), so the safeguard import adds zero dependencies to your server.

Use cases

One mechanism, every stage of the lifecycle.

Behaviour is driven by environment variables, so the same import runs unchanged from your laptop to production.

Local development

Trust on first use

In the default auto mode, the first run registers the server; every later run verifies it. You'll know the moment a refactor, dependency bump or teammate's edit touches the interface you've been testing against.

CI / CD

A verification gate

kiji-safeguard verify server.py exits non-zero on mismatch — drop it into a pipeline step and an unreviewed interface change fails the build before it ships, like a lockfile for your tool surface.

Production

Refuse to start on mismatch

With KIJI_SAFEGUARD_ENFORCE=1, a server whose live interface doesn't match the registry aborts at startup instead of warning. A rug-pulled server never serves a single request.

Teams & audit

A browsable transparency log

The registry (FastAPI + SQLite) ships with a web UI: browse recent registrations, search by name or hash, and inspect the exact interface that was registered — a shared record of what your org has approved.

Quickstart

Running in under a minute.

Pick your path: the CLI for explicit control, or the magic import for zero further code changes.

Path 1 — the CLI

pip install "kiji-safeguard[server]"

# 1. run the registry (FastAPI + SQLite, web UI at /)
kiji-safeguard serve --port 8000

# 2. register a server
kiji-safeguard register mcp_servers/stock_price_server.py

# 3. verify any time — exits non-zero on mismatch
kiji-safeguard verify mcp_servers/stock_price_server.py

Path 2 — the magic import

# add one line to any FastMCP server,
# before or after `mcp` is imported:
import kiji_safeguard.autosign  # noqa: F401

# first run registers, later runs verify
python my_server.py

# production: refuse to start on mismatch
KIJI_SAFEGUARD_ENFORCE=1 python my_server.py

# strict: never auto-register
KIJI_SAFEGUARD_MODE=verify python my_server.py
Configuration — environment variables
Variable Values Default Meaning
KIJI_SAFEGUARD_MODE auto / verify / register / off auto auto verifies and registers unknown servers on first sight; verify never registers; register always publishes; off disables.
KIJI_SAFEGUARD_REGISTRY URL http://127.0.0.1:8000 Registry base URL.
KIJI_SAFEGUARD_ENFORCE 1 / true / … unset Abort startup on verification failure instead of warning.

All diagnostics go to stderr — stdout stays clean for the stdio transport.
